# 校园AR标记预处理与生成规范

> 本规范适用于校园RPG项目的AR标记（基于AR.js NFT图像识别）从拍摄到部署的完整流程。

---

## 1. 概述

### 1.1 AR.js NFT标记工作原理

AR.js 使用 NFT（Natural Feature Tracking，自然特征追踪）技术通过分析图像的纹理特征来识别标记。系统从标记图片中提取大量特征点（边缘、角点、纹理变化），生成唯一的 `.patt` 特征文件。识别时，AR.js 将摄像头画面与已注册的NFT特征进行匹配，匹配成功后叠加虚拟内容。

**特征点密度决定识别效果**：纹理丰富、对比度高的图片能提取更多特征点，识别更快更准；纯色或模糊的图片特征点少，识别困难。

### 1.2 校园标记类型

| 类型 | 示例 | 特征密度 | 预处理难度 |
|------|------|----------|------------|
| 校徽/校标 | 学校Logo、标志性图案 | 高 | ★★☆ |
| 建筑门牌 | 教学楼房间号牌 | 中 | ★☆☆ |
| 图书馆标识 | 书架标签、楼层指示牌 | 中低 | ★★☆ |
| 食堂招牌 | 食堂名称横幅 | 高 | ★☆☆ |
| 公告栏 | 海报、宣传栏 | 极高 | ★★★ |

## 2. 标记拍摄规范

### 2.1 光线要求

- **推荐**：均匀自然光（窗边散射光）或室内柔光LED灯
- **避免**：强光直射（产生镜面反光）、逆光（标记过暗）、单一方向强光（产生硬阴影）
- **校园场景建议**：阴天在户外拍摄，或在室内靠窗位置利用漫反射光线

### 2.2 对焦要求

- 标记边缘和四角必须清晰锐利
- 避免虚焦（失焦导致特征点模糊，识别率大幅下降）
- 手机拍摄时建议手动点击屏幕标记区域进行对焦锁定

### 2.3 构图规范

- 标记占画面 **60%-80%**
- 四角完整可见，不能被裁切
- 预留一定边缘空间（标记周围留5%-10%的背景）
- 背景要求有一定纹理（非纯白/纯黑），但不能过于杂乱

### 2.4 拍摄距离

- 推荐距离：**30cm - 80cm**
- 过近（<20cm）：特征点过于密集，识别不稳定
- 过远（>100cm）：特征点太少，识别率下降

### 2.5 校园场景拍摄示例

| 场景 | 建议 | 说明 |
|------|------|------|
| 校徽（墙面） | 平视拍摄，距离50cm | 确保校徽完整，避免广角畸变 |
| 教学楼门牌 | 平行于墙面拍摄 | 避免倾斜导致透视畸变 |
| 图书馆标识 | 均匀照明下拍摄 | 玻璃框内标识需去掉反光 |
| 食堂招牌 | 侧光补充正面照明 | 招牌通常较高，正面光不足 |
| 公告栏海报 | 镜头与海报平行 | 海报不平整时注意对焦 |

## 3. 标记图片预处理步骤

### 3.1 分辨率与尺寸

- **推荐分辨率**：640 x 640 px 到 1024 x 1024 px
- **必须为正方形**（AR.js要求）
- 如果原图不是正方形，先进行裁剪
- **避免过度放大**（从低分辨率图片拉伸放大会产生噪点，损失特征）

### 3.2 预处理优化

#### 对比度调整

- 提升对比度 15%-25%，使特征点更突出
- 避免过曝（高光溢出丢失细节）和欠曝（暗部死黑）
- 在 Photoshop 中：图像 > 调整 > 对比度，+15~+25
- 在 GIMP 中：颜色 > 亮度-对比度，对比度 +20

#### 锐化处理

- 使用 USM（Unsharp Mask）锐化，半径 1-2px，数量 50%-80%
- 锐化使边缘更清晰，有助于AR.js提取更多角点
- 避免过度锐化（产生光晕和噪点）
- 在 Photoshop 中：滤镜 > 锐化 > USM锐化
- 在 GIMP 中：滤镜 > 增强 > 锐化（智）能锐化

#### 推荐在线工具

| 工具 | 网址 | 适用场景 |
|------|------|----------|
| Squoosh | https://squoosh.app | 浏览器端快速处理，支持对比度和锐化 |
| Figma | https://figma.com | 在线设计，支持精确裁剪和调整 |
| Photoshop Express | https://express.adobe.com | 基础调整，适合移动端 |
| RemoveBG | https://remove.bg | 去除背景（仅当背景杂乱时使用） |

### 3.3 预处理检查清单

完成预处理后，逐项确认：

- [ ] 分辨率在 640x640 - 1024x1024 之间
- [ ] 为正方形，无拉伸变形
- [ ] 对比度适中，高光/暗部细节保留完整
- [ ] 边缘清晰锐利，无明显噪点
- [ ] 背景有一定纹理（非纯色）
- [ ] 文件格式为 PNG（推荐）或 JPG

## 4. NFT特征文件生成步骤

### 4.1 AR.js 官方在线生成器

**访问地址**：https://ar-js-org.github.io/AR.js/ai-maker/marker-training/

**操作步骤**：

1. **上传图片**：点击页面中的 "Select your image" 按钮，上传预处理后的标记图片
2. **输入名称**：在 Pattern name 输入框中填写标记名称（如 `marker_001`、`library_sign`）
3. **开始生成**：点击 "Generate" 按钮，等待服务器处理
4. **等待时间**：通常 30-60 秒，取决于服务器负载和图片大小
5. **下载文件**：生成完成后自动下载两个文件：
   - `.patt` 文件（NFT特征文件，AR识别核心）
   - `.jpg` 文件（标记原图备份）

**注意事项**：
- 生成过程中**不要关闭页面**
- 建议使用 Chrome 或 Edge 浏览器（兼容性最好）
- 如遇服务器超时，等待 5 分钟后重试

### 4.2 替代工具

| 工具 | 网址 | 说明 |
|------|------|------|
| AR.js Marker Creator | https://patosai.github.io/ar-js-tooling/ | 社区维护的替代生成器 |
| NFT Marker Generator（GitHub） | ar-js-org/AR.js 仓库 | 开源自部署方案 |

### 4.3 生成失败排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 上传按钮无响应 | 图片格式不支持 | 转换为 PNG 或 JPG 重新上传 |
| 生成超时 | 服务器负载高 | 等待后重试，或换用替代工具 |
| 下载文件为空 | 浏览器拦截 | 允许浏览器弹窗，或右键另存为 |
| 识别效果差 | 特征点不足 | 提升对比度/锐化，或更换特征更丰富的图片 |
| 生成失败 | 图片不符合要求 | 检查分辨率、尺寸、对比度是否符合规范 |

## 5. 标记文件替换与部署

### 5.1 文件命名规范

每个标记对应一个独立目录：

```
assets/ar/markers/marker_001/
  marker_001.patt     ← NFT特征文件（由生成器下载，去掉.patt后缀在代码中使用）
  marker_001.jpg      ← 标记原图（供打印或在AR场景中展示）
  metadata.json       ← 标记元数据（可选，记录标记配置信息）
```

### 5.2 替换步骤

1. **准备文件**：将 `.patt` 文件（生成器会下载为 `.patt.txt`，需重命名去掉 `.txt` 后缀）和原图放入对应目录
2. **更新配置**：编辑 `js/game/ar/ImageMarker.js` 中的标记配置，设置 `markerNFT` 路径
3. **清除缓存**：按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新
4. **测试验证**：在AR场景中测试识别效果，检查识别距离和准确性

### 5.3 ImageMarker.js 配置示例

```javascript
{
    id: 'marker_001',
    name: '校徽标记',
    description: '扫描校徽解锁校园主线剧情',
    // 演示模式（使用Hiro标记）
    markerUrl: 'assets/ar/markers/marker_001/demo.png',
    markerNFT: null,
    // 正式模式（使用NFT特征文件）
    // markerNFT: 'assets/ar/markers/marker_001/marker_001',  // 去掉.patt后缀
    // markerUrl: 'assets/ar/markers/marker_001/marker_001.jpg',
    isEnabled: true,
    contentType: 'story',
    dialog: '欢迎来到校园开放世界！我是你的向导，点击领取你的新手大礼包！',
    reward: { gold: 100, experience: 50, seed: 'common_knowledge' },
    cooldown: 3600
}
```

### 5.4 ARCore.js 识别参数调优

在 `js/game/ar/ARCore.js` 的 `createSceneDOM()` 中可调整以下参数：

| 参数 | 默认值 | 调整建议 |
|------|--------|----------|
| `minDetectionSize` | 0.15 | 识别距离过远时减小，过近时增大 |
| `maxDetectionSize` | 0.9 | 识别距离过近时减小 |
| `detectionRate` | 30 | 性能允许时增大可提升响应速度 |
| `maxDetectionRate` | 60 | 识别不稳定时减小 |

## 7. 常见问题

### Q: 识别距离太近/太远？
**A**：在 `ARCore.js` 中调整 `minDetectionSize` 和 `maxDetectionSize` 参数。识别距离远时减小 minDetectionSize，识别距离近时减小 maxDetectionSize。

### Q: 识别速度慢，标记出现后需要等待很久才能响应？
**A**：提高 `detectionRate`（建议不超过 30）和 `maxDetectionRate`（建议不超过 60）；同时确保预处理后的标记图片对比度足够高。

### Q: 误识别其他物体为标记？
**A**：确保拍摄时画面中没有二维码、条形码或其他高对比度图案；提高标记图片的对比度和锐化程度，使真实标记的特征点密度远高于干扰物。

### Q: 某些角度无法识别？
**A**：NFT标记支持一定角度的倾斜（建议不超过30度倾斜）；垂直于标记拍摄效果最佳；识别时保持手机与标记平面尽量平行。

### Q: 离线环境下AR功能不可用？
**A**：首次使用时需要联网加载 AR.js 库（从CDN下载约200KB），之后会被Service Worker缓存；如遇缓存失效，重新联网加载一次即可。
